Démarrage rapide
1
🌐
Démarrer XAMPP
Ouvrir XAMPP Control Panel et cliquer Start sur le module Apache. La pastille doit devenir verte. Laisser XAMPP ouvert pendant tout le stream.
2
🤖
Importer les scripts dans Streamer.bot
Dans Streamer.bot → onglet Actions → bouton Import. Charger les fichiers .cs du dossier streamerbot/ du projet. Vérifier que chaque action a bien ses triggers (voir tableau plus bas).
3
🎬
Ajouter la Browser Source dans OBS
Dans OBS → Sources → + → Navigateur.
URL : http://localhost/StreamAlerts/overlay/
Largeur : 1920 · Hauteur : 1080
Cocher « Actualiser le navigateur quand la scène devient active ».
4
⚙️
Configurer les positions et options
Ouvrir la page de configuration pour régler la position, la taille et la visibilité de chaque composant.
Elle peut aussi être utilisée comme OBS Dock : Outils → Docks personnalisés → URL http://localhost/StreamAlerts/config/.
5
✅
Tester l'overlay
Ouvrir l'overlay dans le navigateur et appuyer sur T pour déclencher une alerte de test, ou C pour un message chat. Toutes les touches de test sont listées dans la section Raccourcis ci-dessous.

WriteAlert.cs ⚡ Événement Follow, Sub, Resub, Gift, Raid, Bits, Donation, Channel Points

Mode debug
🔍
Pour diagnostiquer un problème (composant qui ne s'affiche pas, WebSocket qui ne se connecte pas, données qui ne se mettent pas à jour…), ouvrir l'overlay avec ?debug=1 dans l'URL :
http://localhost/StreamAlerts/overlay/?debug=1

Un panneau flottant apparaît avec l'état de la connexion WebSocket, la liste des fichiers JSON en cours de polling et les logs en temps réel. Ce mode n'affecte pas OBS — il ne s'active que dans le navigateur ordinaire.
